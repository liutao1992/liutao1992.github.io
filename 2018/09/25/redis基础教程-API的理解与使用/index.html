通用命令

  • keys

会遍历数据库中所有的键值.该命令的时间复杂度为o(n)

api – keys(一) 演示:

api – keys(二) 演示:

注意: 通常情况下, 在生产环境中我们不建议使用keys 命令. 因为keys命令的时间复杂度O(n).即在redis这样的单线程环境下,如果数据量过大则会造成阻塞.

  • dbsize

计算key的总数. 比如有10个key, 其大小就是10.其时间复杂度为o(1).

api – dbsize 演示

  • exists key

判断一个key是否存在, 这里需要注意, key可以是任意类型.如果数据存在则返回1, 不存在则返回0.其时间复杂度为o(1).

api – exists 演示

  • del key[key…]

删除一个key, 也可以删除多个key. 成功删除则返回1, 失败则返回0.
该命令的时间复杂度为o(1)

api – del key 演示

  • expire key seconds

对一个key设置过期时间, 在该时间段之后就会自动删除.
该命令的时间复杂度为o(1)

  • ttl key

查看key剩余的过期时间

  • persist key

去掉key的过期时间(让key的过期时间不再发生作用)
该命令的时间复杂度为o(1)

api – ttl key 演示

代码注释: 首先我们通过expire命令设置key的有效期限为20秒.当返回值为-2时则表明key已经不存在了.

api – persist key 演示

  • type key

判断一个key的数据类型. 返回的数据类型有 string, hash, list, set, zset(有序集合), none.
该命令的时间复杂度为o(1)

api – type key 演示

注意:因为redis是单线程, 所以我们需要注意以下事项:

  1. 一次只运行一条指令.
  2. 拒绝长(慢)命令,即我们不能使用耗时较多的命令, 比如keys, flushall, flushdb, slow lua script, mutil/exec, operate big value(collection)
